# 🛰️ Спутники и космический мусор — анализ орбит и прогнозирование

> 🇬🇧 [Read in English](README.md)

Двухчастный проект по анализу данных о спутниках и фрагментах космического мусора на орбите Земли. Данные получены из открытой базы [Space-Track.org](https://www.space-track.org) через API и содержат орбитальные параметры тысяч объектов по состоянию на ноябрь 2021 года.

**Часть 1** — разведочный анализ данных и их подготовка. **Часть 2** — построение регрессионных моделей для предсказания орбитального периода.

---

## Датасет

**Источник:** [Kaggle — Satellites and Debris in Earth's Orbit](https://www.kaggle.com/datasets/kandhalkhandeka/satellites-and-debris-in-earths-orbit)  
**Файл:** `space_decay.csv`

### Ключевые признаки

| Признак | Описание |
|---|---|
| `OBJECT_TYPE` | Тип объекта: спутник (PAYLOAD), мусор (DEBRIS), ракета (ROCKET BODY) |
| `INCLINATION` | Наклонение орбиты (градусы) |
| `ECCENTRICITY` | Эксцентриситет орбиты |
| `APOAPSIS` | Высота апогея (км) |
| `PERIAPSIS` | Высота перигея (км) |
| `SEMIMAJOR_AXIS` | Большая полуось орбиты (км) |
| `PERIOD` | Орбитальный период (минуты) — **целевая переменная** |
| `MEAN_MOTION` | Среднее движение (обороты/день) |
| `BSTAR` | Баллистический коэффициент торможения |
| `COUNTRY_CODE` | Страна, запустившая объект |
| `RCS_SIZE` | Размер радарного поперечного сечения: SMALL / MEDIUM / LARGE |
| `LAUNCH_DATE` | Год запуска |
| `NORAD_CAT_ID` | Каталоговый номер NORAD |

## Часть 1 — Разведочный анализ данных (`01_eda.ipynb`)

### Этапы

1. **Загрузка данных** — скачивание через `kagglehub`, удаление технических столбцов, парсинг `EPOCH`
2. **Предобработка** — заполнение пропусков (мода / среднее), поиск дубликатов и физических аномалий
3. **EDA** — гистограммы, тепловая карта корреляций, диаграммы рассеяния, топ-10 стран, типы объектов
4. **Создание признаков** — `ALT_MEAN` (средняя высота орбиты), `ORBIT_CLASS` (LEO / MEO / GEO)
5. **Обработка выбросов** — удаление объектов с `ALT_MEAN > 50 000 км`
6. **Кодирование категорий** — One-Hot Encoding для `RCS_SIZE`, `OBJECT_TYPE`, `SITE`
7. **Отбор признаков** — оставляем признаки с |корреляция с PERIOD| > 0.3
8. **Экспорт** — сохранение очищенного датасета в `orbits.csv`

## Часть 2 — Машинное обучение (`02_modeling.ipynb`)

### Цель
Предсказать орбитальный период (`PERIOD`) объекта по его орбитальным параметрам.

### Этапы

1. **Загрузка данных** — чтение `orbits.csv`, подготовленного в части 1
2. **Разбиение выборки** — 80% обучение / 20% тест, `random_state=42`
3. **Базовые модели** — Линейная регрессия, Случайный лес, Градиентный бустинг
4. **Сравнение моделей** — MAE, RMSE, R² на тесте; график Actual vs Predicted
5. **Подбор гиперпараметров** — `RandomizedSearchCV` для Random Forest и Gradient Boosting
6. **Оценка** — метрики на train и test для диагностики переобучения; итоговая сравнительная таблица
7. **Важность признаков** — какие орбитальные параметры сильнее всего влияют на предсказание
8. **Эксперименты** — разбиение 70/30, нормализация признаков (StandardScaler)
9. **Кросс-валидация** — 5-fold CV RMSE для всех моделей
10. **Анализ чувствительности** — влияние `n_estimators` (RF) и `learning_rate` (GB) на качество
11. **Сохранение моделей** — экспорт лучших моделей через `joblib`

## Выводы

Большинство объектов на орбите — космический мусор (DEBRIS), сосредоточенный на низких орбитах (LEO). Лидеры по числу объектов — Россия/СНГ, США и Китай. Орбитальный период наиболее тесно связан с большой полуосью орбиты, что соответствует третьему закону Кеплера. В части 2 ансамблевые модели (Random Forest, Gradient Boosting) значительно превзошли линейную регрессию, а оптимизированные модели достигли R² близкого к 1.0.
